# Agent Pointers

Rules live in `.ai/rules/` as a single source of truth. Each AI tool gets a thin pointer file that references (not duplicates) the rules.

## Always Generated

These files are always generated regardless of tool selection:

| File               | Purpose                                                       |
| ------------------ | ------------------------------------------------------------- |
| `AGENTS.md` (root) | Universal entry point read by all AI tools                    |
| `.ai/rules/*.md`   | Canonical coding conventions (generated by `10-rules-format`) |
| `.editorconfig`    | Universal editor formatting (if missing)                      |

## Conditional — Based on User Tool Selection

Only generate pointer files for tools the user selected during the Checkpoint.

| File                                     | Tool                    | Size           | Generate when...              |
| ---------------------------------------- | ----------------------- | -------------- | ----------------------------- |
| `.claude/CLAUDE.md`                      | Claude Code             | ~40 lines      | User selected Claude Code     |
| `.github/copilot-instructions.md`        | GitHub Copilot          | ~30 lines      | User selected GitHub Copilot  |
| `.github/instructions/*.instructions.md` | Copilot (path-specific) | ~15 lines each | User selected GitHub Copilot  |
| `.vscode/settings.json`                  | VS Code + Copilot       | ~40 lines      | User selected GitHub Copilot  |
| `.junie/guidelines.md`                   | JetBrains Junie         | ~30 lines      | User selected JetBrains Junie |
| `.cursor/rules/*.mdc`                    | Cursor                  | ~30 lines      | User selected Cursor          |
| `.aider.conf.yml`                        | Aider                   | ~20 lines      | User selected Aider           |

If the user selects no tools, generate only `AGENTS.md`, `.ai/rules/`, and `.editorconfig`.

## .editorconfig (always generated if missing)

Universal editor formatting config. Generate this if the project doesn't already have one. Works with VS Code, IntelliJ, Vim, Emacs, and most editors without plugins.

```ini
# .editorconfig
root = true

[*]
indent_style = {{space or tab}}
indent_size = {{2, 4, etc.}}
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true

[*.md]
trim_trailing_whitespace = false
```

Derive values from the project's existing formatting conventions (observed in code or from formatter config like `.prettierrc`, `pyproject.toml`, etc.).

## .aider.conf.yml (if selected)

Aider reads `.aider.conf.yml` for project configuration. Since rules already live in `.ai/rules/`, generate a minimal config pointing there.

```yaml

# Aider project configuration — coding conventions are in .ai/rules/

read:
  - AGENTS.md
  - .ai/rules/project.md
  - .ai/rules/code-style.md
  - .ai/rules/testing.md
  - .ai/rules/security.md
  - .ai/rules/quality.md

lint-cmd: "{{lint command}}"
test-cmd: "{{test command}}"
auto-lint: true
auto-test: true
```

## .vscode/settings.json (if GitHub Copilot selected)

```json
{
  "github.copilot.chat.codeGeneration.useInstructionFiles": true,
  "github.copilot.chat.reviewSelection.instructions": [
    { "file": ".ai/rules/code-style.md" },
    { "file": ".ai/rules/quality.md" },
    { "file": ".ai/rules/security.md" }
  ],
  "github.copilot.chat.testGeneration.instructions": [
    { "file": ".ai/rules/testing.md" }
  ],
  "github.copilot.chat.commitMessageGeneration.instructions": [
    {
      "text": "Conventional commit format: <type>(<scope>): <subject>. Types: feat, fix, refactor, test, chore, docs, perf, ci. Under 72 chars."
    }
  ],
  "editor.formatOnSave": true,
  "editor.rulers": [100]
}
```

Adjust `editor.tabSize`, `editor.defaultFormatter`, and `editor.rulers` to match the project's code-style conventions.

## .gitignore Additions

Add to the project's `.gitignore` as needed based on selected tools:

```gitignore
# JetBrains user-specific (keep .junie/ tracked if Junie selected)
.idea/*
!.idea/inspectionProfiles/

# Cursor user-specific (keep .cursor/rules/ tracked if Cursor selected)
.cursor/*
!.cursor/rules/
```

All generated directories for selected tools should be committed to version control:

- `tasks/` — task backlog and workflow (commit)
- `.ai/rules/` — coding conventions (commit)
- `.claude/` — Claude pointer (commit, if selected)
- `.github/` — Copilot instructions (commit, if selected)
- `.junie/` — Junie pointer (commit, if selected)
- `.cursor/rules/` — Cursor rules (commit, if selected)
- `.vscode/settings.json` — shared editor settings (commit, if selected)
- `.aider.conf.yml` — Aider config (commit, if selected)
- `.editorconfig` — editor formatting (commit)

If the team prefers to keep task files local, add `tasks/` to `.gitignore` and only commit `tasks/BACKLOG.md` and `tasks/README.md`.
